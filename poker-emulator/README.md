# Poker Emulator

Эмулятор покерного стола для тестирования ботов и тренировки покерных ситуаций с использованием реальных скриншотов.

**Только фронтенд (HTML/CSS/JS) - никакого бэкенда не требуется!**

## Особенность подхода

**Прозрачные области клика**: Вместо того чтобы рисовать свои кнопки поверх скриншота, эмулятор создаёт **невидимые прозрачные области** поверх нативных кнопок на скриншоте. Это даёт два ключевых преимущества:

1. **Визуальная идентичность**: Бот видит оригинальные кнопки на скриншоте, как в реальном покерном клиенте
2. **Точное тестирование**: Бот кликает по тем же координатам, что и на реальном столе

## Как запустить

Просто откройте файл `index.html` в любом браузере:

```bash
# Вариант 1: Двойной клик по index.html
# Вариант 2: Из терминала
open index.html
# или
firefox index.html
# или
```

**Никакого сервера не требуется!** Всё работает локально в браузере.

## Принцип работы

1. **Скриншот** отображается как фон
2. **Невидимые области** (`click-area`) размещаются поверх нативных кнопок на скриншоте
3. Бот "видит" оригинальные кнопки и кликает по ним
4. Клик перехватывается невидимой областью и регистрируется как действие
5. Система проверяет правильность действия и показывает результат

### Режим отладки

В правом верхнем углу есть чекбокс "Показать области". При включении:
- Невидимые области становятся полупрозрачными красными
- Помогает проверить правильность позиционирования

## Текущий тест: AKs

**Сценарий**: Вы на ББ с AKs, оппонент открылся с баттона

**Правильная линия**:
1. **Префлоп**: 3бет (Raise to $0.08)

3. **Турн**: Чек на чек опа
4. **Ривер**: Фолд на бет

## Как добавить новый тест

### 1. Подготовьте скриншоты

Сделайте скриншоты покерного стола для каждой улицы:
- Разрешение должно быть одинаковым для всех скриншотов
- Сохраните в папку `images/`
- Формат: PNG (для прозрачности, если нужно)

### 2. Настройте позиции кнопок через конфигуратор

Откройте `configurator.html` в браузере:

1. **Загрузите скриншот**
2. **Настройте параметры кнопки**:
   - Тип действия (fold/check/call/bet/raise)
   - Текст кнопки
   - Сумма (если есть)
   - Ширина и высота области
3. **Нажмите "Добавить кнопку"**
4. **Перетащите область** на нужное место поверх кнопки на скриншоте
5. **Повторите** для всех кнопок
6. **Скопируйте сгенерированный код**

### 3. Добавьте сценарий в `js/scenarios.js`

```javascript
'my_scenario': {
    name: 'Название теста',
    description: 'Описание ситуации',
    steps: [
        {
            id: 'preflop',
            name: 'Префлоп',
            image: 'images/my_preflop.png',
            instruction: 'Текст инструкции для игрока',
            correctAction: {
                type: 'raise',
                label: 'Raise',
                amount: '$0.08'
            },
            // Вставьте сгенерированный код сюда:
            buttons: [
                {
                    id: 'fold',
                    type: 'fold',
                    label: 'Fold',
                    x: 74.5,      // позиция в процентах
                    y: 88.5,      // позиция в процентах
                    width: 140,   // ширина в пикселях
                    height: 65    // высота в пикселях
                },
                // ... другие кнопки
            ],
            feedback: {
                correct: 'Правильно! Объяснение...',
                incorrect: 'Неправильно. Объяснение...'
            }
        }
        // ... другие шаги
    ]
}
```

### 4. Добавьте в выпадающий список

В `index.html`:

```html
<select id="scenario-select">
    <option value="aks_3bet_cbet">AKs - 3бет -> цбет -> чек -> фолд</option>
    <option value="my_scenario">Название нового теста</option>
</select>
```

### 5. Готово!

Перезагрузите страницу - новый тест появится в списке.

## API для ботов

### Элементы для взаимодействия

Каждая невидимая область имеет атрибуты:
- `id="area-{action}"` - ID области (например, `area-fold`, `area-raise`)
- `data-action="{type}"` - тип действия (`fold`, `check`, `call`, `bet`, `raise`)
- `data-label="{text}"` - текст кнопки
- `data-amount="{amount}"` - сумма ставки (если есть)

### Пример для бота (Python + Selenium)

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
import time

# Открываем локальный файл
driver = webdriver.Chrome()
driver.get('file:///path/to/poker-emulator/index.html')

# Ждём загрузки скриншота
time.sleep(1)

# Находим невидимую область по data-action
fold_button = driver.find_element(By.CSS_SELECTOR, '[data-action="fold"]')

# Кликаем (бот "видит" оригинальную кнопку на скриншоте)
fold_button.click()

# Проверяем результат
time.sleep(0.5)
result = driver.find_element(By.ID, 'result-display')
print(result.text)  # "✓ Правильно! ..." или "✗ Неправильно..."
```

### Пример для бота (Python + PyAutoGUI)

```python
import pyautogui

# Находим окно браузера с эмулятором
# и кликаем по координатам кнопок

# Координаты берутся из конфига:
# x: 74.5%, y: 88.5% относительно игрового экрана

# Кликаем (бот видит оригинальную кнопку на скриншоте)
pyautogui.click(x=894, y=637)  # примерные координаты
```

## Подсказка по позиционированию

Координаты задаются в процентах относительно размера скриншота:
- `x`: горизонтальная позиция центра области (0-100%)
- `y`: вертикальная позиция центра области (0-100%)

Для Natural8 (и похожих клиентов) кнопки обычно находятся:
- По горизонтали: 70-95% (справа)
- По вертикали: 85-92% (внизу)

## Почему это работает для тестирования CV/ботов

Для компьютерного зрения эмулятор **визуально идентичен** реальному клиенту:

- ✅ Используются реальные скриншоты
- ✅ Кнопки на скриншоте видны как есть
- ✅ Нет дополнительных элементов поверх скриншота
- ✅ Невидимые области полностью прозрачны
- ✅ Бот кликает по тем же координатам

Единственное отличие - невидимые области для кликов, но они:
- Полностью прозрачны (`background: transparent`)
- Не имеют визуального представления
- Не мешают распознаванию изображения
